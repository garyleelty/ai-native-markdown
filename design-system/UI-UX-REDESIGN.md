# AI Markdown 编辑器 - UI/UX 全面重新设计方案

## 1. 项目概述

### 1.1 产品定位
AI Markdown 是一款面向开发者和知识工作者的 AI 原生 Markdown 编辑器，核心定位是**内容优先（Content-First）**的生产力工具。产品集成 AI 辅助写作、实时预览、文件管理、知识图谱等现代编辑功能。

### 1.2 设计目标
- **提升沉浸感**：减少视觉噪音，让内容成为焦点
- **强化 AI 感知**：通过设计语言传达智能助手的存在感
- **优化信息层级**：清晰区分导航、编辑、辅助三大功能区
- **统一视觉语言**：建立一致的设计系统，提升品牌识别度
- **增强可访问性**：确保所有用户都能高效使用

### 1.3 当前问题诊断
| 问题类别 | 具体问题 | 影响 |
|---------|---------|------|
| 视觉层级 | 侧边栏与编辑器对比度不足，边界模糊 | 用户难以快速定位功能区 |
| 色彩系统 | 使用纯灰色调，缺乏品牌个性 | 产品辨识度低，视觉单调 |
| 交互反馈 | 按钮状态变化微弱，动效生硬 | 操作确认感弱，体验不流畅 |
| 信息架构 | AI 面板与编辑器关系不清晰 | 用户不理解 AI 如何辅助写作 |
| 移动端 | 响应式断点粗糙，触控目标小 | 移动端体验差，易误触 |
| 排版系统 | 字体层级不明显，行高不统一 | 长文阅读疲劳，扫描效率低 |

## 2. 设计系统（Design System）

### 2.1 设计理念

采用 **"Modern Dark Cinema"** 风格作为核心设计语言：
- **深色优先**：以深色模式为默认，减少眼部疲劳，突出内容
- **电影感层次**：通过微妙的背景层级差异创造空间深度
- **精准克制**：每个视觉元素都有明确的功能目的
- **动态智能**：AI 相关的交互带有微妙的动态反馈

#### 2.6.1 阴影系统

| Token | 值 | 用途 |
|-------|-----|------|
| `--shadow-sm` | `0 1px 2px rgba(0,0,0,0.3)` | 轻微提升 |

#### 2.6.2 环境光效（Ambient Lighting）
- 在深色模式下，使用微妙的径向渐变创造环境深度
- AI 面板区域带有微弱的靛蓝色环境光晕
- 激活的元素带有柔和的顶部边缘高光

### 2.7 动效系统

#### 2.7.1 时间曲线

| 名称 | 曲线 | 用途 |
|------|------|------|
| `--ease-default` | `cubic-bezier(0.4, 0, 0.2, 1)` | 标准过渡（150ms） |
| `--ease-enter` | `cubic-bezier(0, 0, 0.2, 1)` | 元素进入（200ms） |
| `--ease-exit` | `cubic-bezier(0.4, 0, 1, 1)` | 元素退出（150ms） |
| `--ease-spring` | `cubic-bezier(0.16, 1, 0.3, 1)` | 弹性效果（300ms） |
| `--ease-bounce` | `cubic-bezier(0.68, -0.55, 0.265, 1.55)` | 微弹效果（200ms） |

#### 2.7.2 时长规范

| 类型 | 时长 | 用途 |
|------|------|------|
| `--duration-instant` | 0ms | 立即响应 |
| `--duration-fast` | 100ms | 悬停、焦点变化 |
| `--duration-normal` | 150ms | 颜色、透明度过渡 |
| `--duration-slow` | 200ms | 尺寸、位置变化 |
| `--duration-slower` | 300ms | 面板展开/收起 |
| `--duration-enter` | 250ms | 页面/模态框进入 |

#### 2.7.3 微交互规范

**按钮按压**：
- 触发：mousedown / touchstart
- 效果：scale(0.97)，背景色加深
- 时长：100ms
- 恢复：mouseup / touchend 后 150ms 恢复

**切换开关**：
- 滑块移动：200ms spring 曲线
- 背景色变化：150ms default 曲线
- 添加微妙的阴影扩散效果

**面板展开**：
- 高度变化：300ms spring 曲线
- 内容淡入：200ms，延迟 50ms
- 遮罩层：150ms 淡入

**AI 响应指示**：
- 输入时光标旁出现脉冲点
- 脉冲动画：1.5s infinite，ease-in-out
- 响应完成时：脉冲点变为勾选，200ms 过渡

## 3. 信息架构（Information Architecture）

### 3.1 功能模块划分

```
AI Markdown Editor
├── 导航层（Navigation Layer）
│   ├── 全局标题栏（App Header）
│   │   ├── 品牌标识
│   │   ├── 当前文件信息
│   │   └── 全局操作（AI、视图切换、导出、主题）
│   └── 侧边栏（Sidebar）
│       ├── 文件浏览器（File Explorer）
│       ├── AI 配置（AI Config）
│       └── 设置（Settings）
│
├── 内容层（Content Layer）
│   ├── 编辑器（Editor）
│   │   ├── 标签页栏（Tabs）
│   │   ├── 编辑区域（CodeMirror）
│   │   └── 悬浮工具栏（Floating Toolbar）

### 3.3 用户流程图

#### 3.3.1 核心编辑流程

```

    ↓              ↓                ↓            ↓            ↓             ↓
  显示最近文件   文件树导航      实时编辑      智能提示     实时渲染     多格式导出

```

#### 3.3.2 AI 辅助流程

```

         ↓                    ↓              ↓              ↓

```

     ↓              ↓              ↓            ↓            ↓
  系统文件选择器   展开/折叠目录   标签页打开    内容加载      自动保存
  或拖拽文件夹    搜索过滤       或替换当前    到编辑器       或手动保存
```

## 4. 交互设计模式

### 4.1 导航交互

#### 4.1.1 侧边栏行为

**桌面端（≥1024px）**：
- 默认展开，宽度 280px（可拖拽调整 240-400px）
- 点击标题栏「侧边栏」按钮可收起/展开
- 收起后显示为 48px 宽的图标栏
- 拖拽调整宽度时，编辑器区域实时响应

**平板端（768-1023px）**：
- 默认收起，显示图标栏
- 点击图标后展开为悬浮抽屉（300px）
- 点击外部区域或再次点击图标关闭

**移动端（<768px）**：
- 完全隐藏，通过标题栏汉堡菜单触发
- 从左侧滑入的全屏抽屉（85vw）
- 带有半透明黑色遮罩层
- 支持手势左滑关闭

#### 4.1.2 标签页系统

**标签页行为**：
- 新文件在标签页打开
- 标签页显示文件名和关闭按钮
- 未保存更改显示圆点指示器
- 支持拖拽排序
- 溢出时水平滚动，显示渐变遮罩

**标签页状态**：
| 状态 | 视觉表现 |
|------|---------|
| 默认 | 文字 `--text-secondary`，无边框 |
| 悬停 | 背景 `--bg-hover`，文字 `--text-primary` |
| 激活 | 背景 `--bg-base`，底部 2px `--accent-primary` 边框 |
| 未保存 | 文件名后显示 6px 圆点 `--accent-primary` |
| 拖拽中 | 透明度 0.8，轻微放大 scale(1.02) |

### 4.2 编辑器交互

#### 4.2.1 视图模式切换

三种视图模式循环切换：
1. **源码模式（Source）**：纯 CodeMirror 编辑
2. **实时预览（Live）**：编辑+实时渲染并排
3. **预览模式（Preview）**：仅渲染预览

切换方式：
- 标题栏「视图切换」按钮
- 状态栏点击视图模式指示器
- 快捷键（待定义）

切换动画：
- 面板使用 300ms spring 曲线滑入/滑出
- 内容使用 crossfade 过渡

#### 4.2.2 悬浮工具栏

**位置**：编辑器底部居中，距离底部 24px
**行为**：
- 默认显示
- 滚动时保持固定
- 支持拖拽移动位置（记忆偏好）

**工具分组**：
```
[撤销] [重做] | [粗体] [斜体] | [H1] [H2] | [代码] [代码块] [引用] | [列表] [有序列表] | [链接] [图片] | [查找]
```

**交互细节**：
- 按钮 hover：背景 `--bg-hover`，颜色 `--accent-primary`
- 按钮 active：scale(0.95)，100ms
- 分组间 1px 分隔线 `--border-default`
- 工具提示：上方显示，150ms 延迟

#### 4.2.3 选中文字交互

**选中后行为**：
- 显示浮动操作条（Floating Action Bar）
- 位置：选中区域上方 8px
- 操作：AI 润色、AI 续写、加粗、斜体、复制

**浮动操作条动画**：
- 进入：从下方 8px 滑入 + 淡入，200ms spring
- 离开：向上 8px 滑出 + 淡出，150ms

### 4.3 AI 面板交互

#### 4.3.1 面板展开/收起

**触发方式**：
- 标题栏 AI 按钮
- 快捷键（Cmd/Ctrl + Shift + A）
- 选中文字后的「AI 润色」操作

**展开动画**：
- 从底部向上滑入，300ms spring
- 高度从 0 增长到默认 280px
- 编辑器区域同步压缩

**收起动画**：
- 向下滑出，200ms ease-out
- 高度压缩到 0
- 编辑器区域恢复

#### 4.3.2 AI 对话交互

**消息气泡**：
- 用户消息：右对齐， `--bg-surface` 背景
- AI 消息：左对齐， `--bg-elevated` 背景
- 系统消息：居中， `--text-muted`，小字

**流式响应**：
- 显示「AI 思考中...」脉冲指示器
- 文字逐字打出效果（打字机效果）
- 响应完成后显示操作按钮组

**快捷操作**：
- 每条 AI 消息下方显示：「插入」「替换」「复制」「重新生成」
- 按钮紧凑排列，图标+文字

### 4.4 文件树交互

#### 4.4.1 文件操作

**右键菜单（Context Menu）**：
- 新建文件
- 新建文件夹
- 重命名
- 删除（带确认）
- 复制路径

**拖拽操作**：
- 文件可拖拽排序
- 支持拖拽到文件夹内
- 拖拽时显示半透明预览

#### 4.4.2 搜索交互

**搜索框**：
- 位于文件树顶部
- 聚焦时边框变为 `--accent-primary`
- 实时过滤文件列表
- 支持内容搜索（切换模式）

**搜索结果**：
- 文件名匹配：高亮匹配部分
- 内容匹配：显示匹配行预览
- 点击跳转并高亮匹配位置

### 4.5 导出交互

**导出菜单**：
- 点击标题栏导出按钮触发
- 下拉菜单显示：Markdown、HTML、PDF（未来）
- 菜单项带图标和描述

**导出过程**：
- 显示进度指示器（如需要）
- 完成后显示 Toast 通知
- 自动触发下载

## 6. 页面设计

### 6.1 空状态（Empty State）

**场景**：未打开文件夹时

**设计**：
- 居中显示大图标（48px， `--text-muted`）
- 标题："开始写作"，20px， `--text-primary`

- 背景：微妙的点阵或网格纹理

**动画**：
- 图标：轻微的浮动动画（translateY ±4px，3s infinite）
- 文字：渐入，依次延迟 100ms
- 按钮：从下方滑入，延迟 200ms

## 8. 可访问性（Accessibility）

### 8.1 键盘导航

**全局快捷键**：
| 快捷键 | 功能 |
|--------|------|
| Cmd/Ctrl + N | 新建文件 |
| Cmd/Ctrl + O | 打开文件夹 |
| Cmd/Ctrl + S | 保存文件 |
| Cmd/Ctrl + Shift + A | 切换 AI 面板 |
| Cmd/Ctrl + Shift + E | 切换侧边栏 |
| Cmd/Ctrl + B | 切换文件浏览器 |
| Cmd/Ctrl + Shift + F | 全局搜索 |
| Cmd/Ctrl + 1/2/3 | 切换视图模式 |

**焦点管理**：
- 所有交互元素可通过 Tab 聚焦
- 焦点环：2px `--accent-primary`，offset 2px
- 焦点顺序遵循视觉顺序
- 模态框打开时，焦点限制在模态内

### 8.2 屏幕阅读器

**ARIA 标签**：
- 图标按钮必须有 `aria-label`
- 当前页面/标签使用 `aria-current`
- 展开/收起状态使用 `aria-expanded`
- 模态框使用 `aria-modal` 和 `role="dialog"`

**动态内容**：
- Toast 通知使用 `aria-live="polite"`
- AI 响应使用 `aria-live="assertive"`
- 加载状态使用 `aria-busy`

### 8.3 色彩对比度

| 元素 | 对比度要求 | 实际对比度 |
|------|-----------|-----------|
| 主要文字 | ≥ 4.5:1 | 14.5:1（白 on 黑） |
| 次要文字 | ≥ 4.5:1 | 8.2:1 |
| 辅助文字 | ≥ 3:1 | 5.1:1 |
| 强调色按钮 | ≥ 4.5:1 | 7.8:1 |
| 禁用文字 | 无要求 | 2.8:1 |

### 8.4 动效偏好

**减少动效（prefers-reduced-motion）**：
- 所有过渡时间设为 0ms
- 禁用浮动、脉冲等装饰动画
- 保留必要的功能动画（如面板展开）
- 使用即时状态切换替代渐变

## 附录 A：设计决策记录

### A.1 为什么选择深色优先？

1. **产品定位**：Markdown 编辑器面向开发者，深色模式是默认预期
2. **内容聚焦**：深色背景让文字内容更突出，减少视觉干扰
3. **AI 感知**：深色模式更容易展示 AI 相关的光效和动态
4. **行业趋势**：Notion、Obsidian、VS Code 等工具均采用深色优先

### A.2 为什么选择 Inter + JetBrains Mono？

1. **Inter**：现代无衬线字体，专为屏幕阅读优化，字重丰富
2. **JetBrains Mono**：专为代码设计，连字支持，开发者熟悉
3. **搭配效果**：UI 清晰现代，编辑器专业高效
4. **性能**：Google Fonts 支持，加载快速

### A.3 为什么采用底部 AI 面板？

1. **上下文保持**：AI 辅助是编辑的延伸，底部位置保持编辑上下文
2. **空间效率**：桌面端宽度充裕，底部面板不占用水平空间
3. **可预测性**：类似 VS Code 终端、Notion AI 的熟悉模式
4. **响应式**：移动端可全屏展开，桌面端可调整高度
