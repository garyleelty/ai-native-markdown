# 文件管理功能设计文档

## 背景

当前 AI Native Markdown 编辑器只能"打开文件夹"和"选择已有文件"，用户无法：
- 新建 Markdown 文件
- 新建文件夹
- 重命名文件/文件夹
- 删除文件/文件夹

这导致产品停留在"玩具"阶段，无法成为日常写作工具。

## 目标

将文件树从扁平列表升级为**可展开的树形结构**，并提供完整的文件 CRUD 操作。

## 方案概述

采用**树内直接编辑**的交互风格（Obsidian 风格），所有操作即时反馈，无需弹窗。

## 详细设计

### 1. 文件树数据结构

将当前扁平的 `FileItem[]` 升级为树形节点：

```typescript
interface TreeNode {
  id: string        // 文件/文件夹路径
  name: string      // 显示名称
  path: string      // 完整路径
  isDirectory: boolean
  isMarkdown: boolean
  children?: TreeNode[]   // 子节点（仅文件夹）
  isExpanded?: boolean    // 展开状态
  isEditing?: boolean     // 是否处于重命名/新建输入状态
  isNew?: boolean         // 是否为新创建的节点（未确认）
}
```

### 2. 文件树 UI 结构

```
EXPLORER  [新建文件] [新建文件夹]
├── 📁 workspace/              ← 根文件夹（可展开/折叠）
│   ├── 📄 README.md           ← 文件（可点击打开）
│   ├── 📁 docs/               ← 子文件夹（可展开）
│   │   ├── 📄 guide.md
│   │   └── 📄 api.md
│   ├── 📄 笔记.md
│   └── [输入文件名...]        ← 新建时的内联输入框
```

### 3. 操作交互

#### 3.1 新建文件
1. 用户点击头部"新建文件"按钮（文件+图标）
2. 在当前选中文件夹（或根文件夹）下方出现一个内联输入框
3. 输入框自动聚焦，placeholder 为"新文件名.md"
4. 用户输入文件名，按 `Enter` 确认
5. 调用 Tauri `create_file` 命令创建空文件
6. 刷新该文件夹的子节点列表
7. 自动选中新文件并打开编辑

#### 3.2 新建文件夹
1. 用户点击头部"新建文件夹"按钮（文件夹+图标）
2. 在当前选中文件夹下方出现内联输入框
3. placeholder 为"新文件夹名"
4. 按 `Enter` 确认，调用 `create_dir`
5. 刷新文件夹列表，新文件夹默认展开

#### 3.3 重命名
1. 右键点击文件/文件夹 → 选择"重命名"
2. 文件名变为可编辑输入框，预填充当前名称
3. 按 `Enter` 确认，调用 `rename_file`
4. 按 `Esc` 取消

#### 3.4 删除
1. 右键点击文件/文件夹 → 选择"删除"
2. 显示确认提示（文件直接删除，文件夹需二次确认）
3. 调用 `delete_file` 或 `delete_dir`
4. 刷新列表

### 4. 右键菜单设计

自定义右键菜单（避免浏览器默认菜单）：

```
┌─────────────────┐
│ 重命名          │
│ 删除            │
├─────────────────┤
│ 新建文件        │
│ 新建文件夹      │
└─────────────────┘
```

- 菜单出现在鼠标右键位置
- 点击外部或按 Esc 关闭
- 深色/浅色主题自适应

### 5. 子文件夹展开机制

- 点击文件夹名称或左侧 chevron 图标切换展开/折叠
- 展开时：调用 `read_dir` 加载该文件夹下的内容
- 展开状态缓存在组件内部（不持久化）
- 仅展示 `.md` / `.markdown` 文件和子文件夹（保持现有过滤逻辑）

### 6. 错误处理

| 场景 | 处理方式 |
|------|---------|
| 文件名已存在 | 输入框下方显示红色提示"文件名已存在" |
| 文件名包含非法字符 | 实时校验，提示"文件名包含非法字符" |
| 创建失败（权限等） | Toast 提示错误信息 |
| 删除文件夹非空 | 后端 `remove_dir_all` 会递归删除，前端二次确认即可 |

### 7. 键盘快捷键

| 快捷键 | 功能 |
|--------|------|
| `Enter` | 确认新建/重命名 |
| `Esc` | 取消新建/重命名，关闭右键菜单 |
| `F2` | 对选中项触发重命名 |
| `Delete` | 对选中项触发删除（需确认） |

### 8. 与现有代码的整合

#### 改动文件：
- `src/components/Sidebar.vue` — 主要改动：重构文件树为递归组件
- `src/components/FileTree.vue` — 可复用或合并到 Sidebar
- `src/App.vue` — 可能需要调整文件打开逻辑

#### 后端命令（已存在，无需新增）：
- `create_file(path)` — 创建空文件
- `create_dir(path)` — 创建文件夹
- `rename_file(old_path, new_path)` — 重命名
- `delete_file(path)` — 删除文件/文件夹
- `read_dir(path)` — 读取目录内容

## 视觉设计

- 新建输入框：与文件项同高，边框高亮（accent 色），placeholder 灰色
- 右键菜单：圆角卡片，阴影，hover 时背景色变化
- 展开/折叠动画：chevron 旋转 90°，子项淡入
- 保持现有暗色/亮色主题变量一致性

## 范围控制

**本次不做：**
- 文件拖拽排序
- 跨文件夹拖拽移动
- 文件搜索（已有独立搜索）
- 文件内容搜索

## 验收标准

- [ ] 点击"新建文件"按钮可在树内创建新 `.md` 文件
- [ ] 点击"新建文件夹"按钮可在树内创建新文件夹
- [ ] 右键文件/文件夹可重命名
- [ ] 右键文件/文件夹可删除（有确认）
- [ ] 文件夹可展开/折叠，展示子内容
- [ ] 新建/重命名输入框支持 Enter 确认、Esc 取消
- [ ] 错误场景有明确提示
- [ ] 暗色/浅色主题下视觉一致
