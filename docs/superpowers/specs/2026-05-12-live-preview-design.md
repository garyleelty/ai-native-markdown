## 1. 产品概述

### 1.1 功能定位

Live Preview（实时预览模式）是一种混合编辑体验，让用户在写作时能立刻看到排版后的效果，同时保留对 Markdown 源码的控制力。官方将其称为"所见即所得"（WYSIWYG）模式。

### 1.2 核心目标

- 消除编辑与预览之间的上下文切换成本
- 保持 Markdown 源码的可控性和可移植性
- 提供与 Obsidian 一致的编辑体验

### 1.3 三种视图模式

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| **源码模式 (Source)** | 纯文本编辑，显示所有原始语法 | 精确调整复杂语法 |
| **实时预览 (Live Preview)** | 混合编辑，光标附近显示源码，其他区域渲染 | 日常写作、快速记录 |
| **阅读模式 (Reading)** | 仅查看，完全渲染，不能编辑 | 最终审阅、阅读 |

## 2. 架构设计

### 2.1 技术基石

基于 CodeMirror 6 的 Decoration 系统实现：

```
┌─────────────────────────────────────────────┐
│              CodeMirror 6 Editor              │
│  ┌───────────────────────────────────────┐  │
│  │  ViewPlugin: livePreviewPlugin        │  │
│  │  ┌─────────────────────────────────┐  │  │
│  │  │  1. 监听光标位置变化              │  │  │
│  │  │  2. 解析 Markdown 语法树          │  │  │
│  │  │  3. 生成 Decoration 集合          │  │  │
│  │  │  4. 非光标行 → 渲染视觉效果       │  │  │
│  │  │  5. 光标行 → 显示原始源码         │  │  │
│  │  └─────────────────────────────────┘  │  │
│  └───────────────────────────────────────┘  │
│  ┌───────────────────────────────────────┐  │
│  │  Decoration System                    │  │
│  │  - Decoration.replace() 替换语法      │  │
│  │  - Decoration.widget()  嵌入组件      │  │
│  │  - Decoration.mark()    样式标记      │  │
│  └───────────────────────────────────────┘  │
└─────────────────────────────────────────────┘
```

### 2.2 核心运作机制

编辑器同时扮演两种角色：

1. **底层仍是源码**：编辑器底层维护 Markdown 源码，所有操作实际都在编辑这份源码
2. **上层是"贴纸"**：通过 Decoration 系统，在分析源码后生成覆盖层，将 Markdown 语法替换为视觉效果

### 2.3 智能"隐身"与"现身"

| 场景 | 行为 |
|------|------|
| 光标离开后 | 自动将 Markdown 语法渲染为视觉效果，展示最终排版样式 |
| 光标靠近/进入 | 移除覆盖的 Decoration，源码重新"现身"，便于直接编辑 |
| 点击已渲染元素 | 计算对应源码位置，将光标跳转过去并进入编辑模式 |

## 3. 功能设计

### 3.1 视图模式切换

**状态变更**：
- `ViewMode` 从 `'source' | 'preview'` 扩展为 `'source' | 'live' | 'preview'`
- 在 `editorStore` 中支持三种模式切换
- 状态栏显示当前模式，点击可循环切换

**切换行为**：
- Source → Live：编辑器保持内容，启用 Decoration 系统
- Live → Source：移除所有 Decoration，显示纯源码
- Live → Preview：切换到只读预览面板（现有逻辑）

### 3.2 渲染元素优先级

#### P0 - 基础排版（第一阶段实现）

| 元素 | 渲染方式 | 交互行为 |
|------|----------|----------|
| 粗体 `**text**` | 字体加粗，隐藏 `**` | 点击进入编辑 |
| 斜体 `*text*` | 字体倾斜，隐藏 `*` | 点击进入编辑 |
| 删除线 `~~text~~` | 文字删除线，隐藏 `~~` | 点击进入编辑 |
| 行内代码 `` `code` `` | 等宽字体、背景色，隐藏 `` ` `` | 点击进入编辑 |
| 链接 `[text](url)` | 隐藏语法，显示带下划线的文字 | 点击跳转/编辑 |
| 图片 `![alt](url)` | 隐藏语法，直接显示图片 | 点击进入编辑 |
| 标题 `# H1` ~ `###### H6` | 隐藏 `#`，应用对应字号 | 点击进入编辑 |
| 引用块 `> text` | 隐藏 `>`，左侧边框+背景色 | 整行编辑 |
| 无序列表 `- item` | 隐藏 `-`，显示圆点 | 整行编辑 |
| 有序列表 `1. item` | 隐藏 `1.`，显示数字 | 整行编辑 |
| 水平线 `---` | 隐藏语法，显示分割线 | 整行编辑 |

#### P1 - 增强元素（第二阶段实现）

| 元素 | 渲染方式 | 交互行为 |
|------|----------|----------|
| 任务列表 `- [ ] task` | 显示可交互复选框 | 点击勾选/取消 |
| 表格 `\| col \| col \|` | 渲染为表格样式 | 点击进入编辑 |
| 代码块 ` ```lang ` | 语法高亮块 | 点击进入编辑 |
| Wiki 链接 `[[note]]` | 显示为内部链接样式 | 点击跳转/编辑 |

#### P2 - 复杂元素（第三阶段实现）

| 元素 | 渲染方式 | 交互行为 |
|------|----------|----------|
| KaTeX 公式 `$...$` / `$$...$$` | 渲染数学公式 | 点击进入编辑 |
| Mermaid 图表 | 渲染为 SVG 图表 | 点击进入编辑 |

### 3.3 光标交互行为

**整行切换策略**：

1. 光标所在行（cursor line）及其相邻行显示原始 Markdown 源码
2. 其他所有行应用 Decoration 渲染为视觉效果
3. 光标移动时，实时重新计算 Decoration 范围

**点击编辑策略**：

1. 用户点击已渲染的元素（如链接、图片）
2. 系统计算该元素在源码中的位置范围
3. 将光标设置到对应位置，并切换为源码显示
4. 用户可直接编辑该元素的 Markdown 语法

**防抖优化**：
- 光标位置变化触发 Decoration 更新时，使用 50ms 防抖
- 避免快速移动光标时频繁重计算

## 4. 模块设计

### 4.1 文件结构

```
src/
├── components/
│   └── editor/
│       ├── Editor.vue              # 现有编辑器，扩展模式支持
│       └── LivePreviewEditor.vue   # Live Preview 专用编辑器封装
├── composables/
│   └── useLivePreview.ts           # Live Preview 核心逻辑组合式函数
├── extensions/
│   ├── livePreview/                # CodeMirror 6 扩展示例
│   │   ├── index.ts                # 统一导出
│   │   ├── plugin.ts               # 主 ViewPlugin
│   │   ├── decorations.ts          # Decoration 生成逻辑
│   │   ├── parsers.ts              # Markdown 语法解析
│   │   └── widgets/                # 自定义 Widget 组件
│   │       ├── LinkWidget.ts
│   │       ├── ImageWidget.ts
│   │       ├── TaskListWidget.ts
│   │       └── TableWidget.ts
│   └── index.ts
```

### 4.2 核心模块职责

| 模块 | 职责 | 依赖 |
|------|------|------|
| `livePreviewPlugin` | CodeMirror ViewPlugin，监听状态变化，协调渲染 | `@codemirror/view`, `@codemirror/state` |
| `decorations.ts` | 根据语法树和光标位置生成 Decoration 集合 | `parsers.ts` |
| `parsers.ts` | 解析 Markdown 源码，识别各元素的位置和类型 | 正则表达式 + 简单语法分析 |
| `widgets/*.ts` | 复杂元素的自定义 Widget 实现（链接、图片等） | `@codemirror/view` |

### 4.3 数据流

```
用户输入/光标移动
    │
    ▼
┌──────────────────┐
│ CodeMirror State │
│ Update           │
└──────────────────┘
    │
    ▼
┌──────────────────┐
│ livePreviewPlugin│
│ - 获取光标位置   │
│ - 获取文档内容   │
└──────────────────┘
    │
    ▼
┌──────────────────┐
│ parsers.ts       │
│ - 解析 Markdown  │
│ - 识别元素范围   │
└──────────────────┘
    │
    ▼
┌──────────────────┐
│ decorations.ts   │
│ - 过滤光标行     │
│ - 生成 Decoration│
└──────────────────┘
    │
    ▼
┌──────────────────┐
│ CodeMirror View  │
│ - 应用 Decoration│
│ - 渲染视觉效果   │
└──────────────────┘
```

### 5.2 Live Preview 视觉效果

**行内元素样式**：

| 元素 | 视觉效果 |
|------|----------|
| 粗体 | `font-weight: 600` |
| 斜体 | `font-style: italic` |
| 删除线 | `text-decoration: line-through` |
| 行内代码 | 等宽字体、圆角背景、内边距 |
| 链接 | 主题色、下划线、hover 效果 |
| 图片 | 圆角、最大宽度 100%、阴影 |

**块级元素样式**：

| 元素 | 视觉效果 |
|------|----------|
| 标题 H1 | 大字号、底部边框 |
| 标题 H2 | 较大字号、底部边框 |
| 引用块 | 左侧主题色边框、背景色 |
| 无序列表 | 圆点标记 |
| 有序列表 | 数字标记 |
| 水平线 | 细线分隔 |
| 代码块 | 深色背景、语法高亮、圆角 |

## 6. 性能设计

### 6.1 优化策略

| 策略 | 实现方式 |
|------|----------|
| 增量更新 | 仅重新计算光标附近 5 行的 Decoration，其他行复用缓存 |
| 防抖处理 | 光标移动 50ms 防抖后再更新 Decoration |
| 虚拟渲染 | 只渲染可视区域内的 Decoration |
| 缓存语法树 | 文档未变更时复用上一次的解析结果 |

### 6.2 性能目标

| 指标 | 目标值 |
|------|--------|
| 光标移动响应 | < 16ms |
| 大文件（10MB）| 流畅编辑，无明显卡顿 |
| 初始加载 | < 100ms |

## 7. 错误处理

### 7.1 降级策略

| 场景 | 处理策略 |
|------|----------|
| Decoration 渲染失败 | 降级为显示原始 Markdown 源码 |
| 复杂元素解析失败 | 保持原始语法显示，不应用 Decoration |
| 性能瓶颈检测 | 自动禁用 Live Preview，提示用户切换至 Source 模式 |

### 7.2 边界情况

- **空文档**：正常显示，无 Decoration
- **纯代码块文档**：代码块内不应用行内 Decoration
- **表格内部**：表格单元格内支持行内元素渲染
- **嵌套元素**：如粗体中的斜体，正确嵌套渲染

### 10.1 术语表

| 术语 | 说明 |
|------|------|
| Decoration | CodeMirror 6 的装饰系统，用于在编辑器中渲染自定义视觉效果 |
